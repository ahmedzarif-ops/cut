# Campaign Brief (intake)

Fill this with the owner at the start of a run. Pull defaults from the KBs first; ask
only about real gaps. Never invent research. One brief can cover one push (one offer, one
or two pages, one or two channels).

- Objective: <what this push is for, e.g. "drive completed diagnostics for segment A + B" or "fill the founding cohort for the recurring tier">
- Target offer / tier: <free diagnostic / entry paid offer / recurring offer / high-touch consult>
- Target page(s): <the destination landing page(s) or asset>
- Segment focus: <which personas and segments; default from segmentation.md>
- Awareness stage(s): <Unaware / Problem / Solution / Product / Most; default Problem-to-Solution for cold>
- Beachhead channels: <default the demand channel + search; note any add>
- Budget band: <rough monthly spend, or "strategy only, no spend yet">
- Geo: <national by targeting, or a specific metro>
- Constraints: <anything off-limits, brand notes, timing>
- Deadline / seasonality: <any real seasonal trigger or launch-window timing>

Notes:
- Default channels = the demand channel + search (the beachhead). Confirm before adding
  more.
- Default proof posture = founder/authority-led + product-as-proof + sourced stats + any
  real guarantee. No fabricated proof.
- If the brief implies organic or lifecycle work, the strategy will route it to the
  existing engine (blog-engine / social-engine / email-marketing / in-product follow-up),
  not build it here.
