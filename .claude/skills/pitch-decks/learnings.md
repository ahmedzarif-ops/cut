# Vertical Pitch-Deck Engine - Learnings

The running registry of pitch-deck runs. Append one entry per run so the next run starts from what
we already learned.

## How to use this file

After each run, append an entry under Runs with: the date, the brief (which segments + leads),
what the decks bet on, what shipped (how many decks), the compliance result, what worked or
surprised, and what to change next time. Keep entries short and skimmable. Cross-reference your ICP
research doc (`company.yml -> icp.research_doc`) for the segment truth.

Entry shape:

```
### YYYY-MM-DD - <short title>
- Brief: <segments / leads / scope>
- Bet: <the single biggest content bet>
- Shipped: <how many decks / what>
- Compliance: <PASS / flags fixed>
- Worked / surprised: <notes>
- Change next time: <notes>
```
