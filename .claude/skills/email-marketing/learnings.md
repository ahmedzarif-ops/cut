# Email + Lifecycle Engine - Learnings

The running registry of lifecycle-email runs. Append one entry per run so the next run starts from
what we already learned.

## How to use this file

After each run, append an entry under Runs with: the date, the brief (run scope, programs,
segments), what the lifecycle map bet on, what copy shipped, the compliance result, what worked or
surprised, and what to change next time. Keep entries short and skimmable. Cross-reference your
market-radar / demand-signal feed under `kb_dir` for demand context and the `paid-media`
`learnings.md` for funnel context.

Entry shape:

```
### YYYY-MM-DD - <short title>
- Brief: <run scope / programs / segments>
- Map bet: <the single biggest lifecycle bet>
- Shipped: <which sequences / libraries / how many emails>
- Compliance: <PASS / flags fixed>
- Worked / surprised: <notes>
- Change next time: <notes>
```
