# Email Program Brief (intake)

Fill this with the owner at the start of a run. Pull defaults from the KBs and `company.yml`
first; ask only about real gaps. Never invent research. One brief covers one run (a FOUNDATION run
or one named program).

- Run scope: <FOUNDATION (programs A, D, F, G) | a named program from lifecycle-programs.md>
- Objective: <what this run is for, e.g. "convert free-thing completers to the entry offer">
- In-scope programs: <A-L; default to the run scope above>
- Target segments: <funnel-stage / buyer-reality values; default from segmentation.md>
- Segment focus: <all-segments core + light per-segment variants, or a specific segment>
- Awareness stage(s): <Unaware / Problem / Solution / Product / Most; default per program>
- Primary weak-factor branches (program D only): <all factors, or a subset>
- ESP target: <platform-neutral, no wiring | the platform from company.yml -> integrations.esp>
- Constraints: <brand notes, off-limits, timing, current policies (pricing window open?)>
- Seasonality: <any real seasonal trigger to use as honest urgency>

Notes:
- Default proof posture = founder-led + product-as-proof + sourced stats + any real guarantee
  (honest proof only). No fabricated proof.
- Pricing always from `company.yml` -> `offer.pricing_notes` (never a stale figure; see guardrails.md).
- If the brief implies paid, blog, or social work, route it to the matching engine, not here.
